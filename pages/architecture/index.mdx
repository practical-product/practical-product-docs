# 아키텍처 개요

Practical Product는 백엔드와 프론트엔드를 별도 저장소로 분리한
**모노레포 성격의 멀티 레포 구조**입니다.

practical-product/
├─ backend (Spring Boot)
├─ frontend (Next.js App Router)
└─ docs 

## 설계 원칙

1. **도메인 경계 명확화** — 백엔드는 member/product/category/order/payment로
   모듈을 나누고, 각 모듈은 `-api`(공개 인터페이스)와 `-core`(내부 구현)로
   다시 분리했습니다. 다른 모듈은 `-api`만 참조할 수 있어 내부 구현 변경이
   외부에 영향을 주지 않습니다.
2. **결제는 이벤트로 분리** — 결제 승인/취소/실패는 도메인 이벤트로 발행되고,
   Outbox 패턴으로 유실 없이 전파됩니다.
3. **프론트는 기능 단위로 분리** — `features/`, `shared/`, `widgets/`로
   나눠 화면(app router 페이지)과 로직을 분리했습니다.
