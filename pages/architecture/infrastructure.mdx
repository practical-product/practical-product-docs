# 인프라 구성

Docker 기반 컨테이너 배포와 GitHub Actions CI/CD, 그리고
Prometheus·Grafana·Loki·Tempo로 구성된 모니터링 스택을 운영합니다.

## CI/CD 파이프라인

- `deploy` 브랜치로 병합되면 GitHub Actions가 자동으로 빌드·배포를 트리거
- Docker 이미지 빌드 시 라이브러리 버전을 고정값으로 관리해 재현성 확보
- EC2 배포 단계에서 Docker Hub 로그인 → 이미지 Pull → 컨테이너 재기동 순서로 진행
- 코드 리뷰 봇이 PR마다 자동으로 빌드·테스트를 실행해 배포 전 문제를 조기에 발견

## 모니터링 & 관측성

- **Prometheus + Grafana**: 비즈니스 지표와 RED(Rate·Error·Duration) 대시보드
- **Loki**: 구조화된 로그 수집, 상관관계 ID(Correlation ID)로 요청 단위 추적
- **Tempo**: 분산 트레이싱으로 요청이 각 모듈을 거치는 흐름 파악
- **Discord 알림**: 결제 실패율 등 주요 지표 임계치 초과 시 실시간 알림
- 로컬 개발 환경에서도 동일한 Prometheus + Grafana 스택을 `bootRun`에 통합해
  서버와 함께 자동 기동
